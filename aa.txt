あなたは「CV最優先のBtoB SaaS LP」を作る世界トップクラスのプロダクトデザイナー兼フロントエンドエンジニアです。
目的は BizFlo の問い合わせ（導入相談 / 資料請求）のCVを最大化すること。見た目・コピー・導線・速度・SEO・信頼性をすべて最高水準で仕上げてください。
“AIっぽさ”は絶対に出さず、日本の中小企業経営者/管理部門/営業責任者に刺さる自然な言葉で書くこと。

# 0) BizFlo概要（LPで伝えるべき事実）
- BizFloは、顧客管理・案件管理・タスク管理・KPI管理・売上管理・会計管理・受発注管理を統合した業務OS（ERP的統合SaaS）。
- 価格：月額 29,800円（スタンダード想定）。エンタープライズ/個別構築も用意（問い合わせ導線）。
- 強み：
  - 複数ツールの分断を解消し「二重入力・集計・月次締め」を削減
  - 案件と売上/粗利/稼働がつながり、経営判断が速くなる
  - KPIダッシュボードや稼働分布など“現場〜経営”まで一気通貫
  - 会社ごとの運用に合わせて柔軟に育つ（カスタム項目/運用設計/伴走）
- 競合文脈：Salesforce / freee / 楽々販売 など「点の最適」になりやすいツール群の良いところ取りを“1つに統合”。

※数値実績（例：◯◯時間削減、ROI等）は、断定せず「試算」「例」として扱い、注意書き（※条件により変動）を明記すること。

# 1) 技術スタック（これで実装）
- Next.js (App Router) + TypeScript
- Tailwind CSS
- shadcn/ui（Button, Card, Accordion, Tabs, Dialog, Input 等）
- Framer Motion（過剰にならない上品なアニメ）
- 画像はNext Image最適化、フォントは Noto Sans JP（Google Fonts）
- SEO: metadata / OGP / JSON-LD（SoftwareApplication, FAQPage, Organization）
- パフォーマンス: Lighthouse 95+を狙う（CLS/LCP最適化、不要JS排除）
- デプロイ前提: Vercel（ただしコードは汎用で）

# 2) 成果物（必須）
1) 完全なコードベース一式（実行できる）
2) 1ページ完結のLP（/）として作成
3) 問い合わせCTA導線（2種類）
   - 「導入相談する」（メイン）
   - 「資料請求」（サブ）
   ※フォームはフロントだけでもいいが、バリデーションと送信成功UIまで作ること。送信先は仮でOK（例：/api/lead を用意してダミー保存 or consoleログ）。後で差し替えできる設計にする。
4) “勝つための仕掛け”を実装
   - スクロール追従の固定CTA（ヘッダー + モバイル下部バー）
   - ROI/削減効果の「入力式」ミニ計算機（従業員数/時給/現在の月次作業時間など→削減見込みを表示）
   - 競合比較表（複数ツール運用 vs BizFlo）
   - ユースケース（経営者/営業責任者/管理部門）タブ切替
   - FAQ（アコーディオン）
   - 信頼セクション（セキュリティ/運用支援/導入フロー）
5) コンテンツは data/content.ts または content.json のように分離し、後で編集しやすくする
6) READMEにセットアップ手順、環境変数、フォーム差し替え箇所を明記

# 3) デザイン/トーン（必須）
- 「AI生成感」禁止。シンプルでセンスが良い、日本のSaaSらしい信頼感（青・白基調、アクセント1色程度）。
- 余白・タイポ・階層が美しい。1画面目で“価値”が一瞬で伝わる。
- 文章は短く強く。専門用語は使いすぎない。
- どこを見てもCTAが自然に視界に入り、押したくなる。
- スマホ最適化：親指導線でCVを最優先。

# 4) LP構成（この順で実装）
A. Hero（最重要）
- H1案（複数用意して最良を採用）：
  例）「月3万円で、会社の“いま”が全部わかる。」
  例）「顧客・案件・請求・会計を、ひとつに。」
- サブコピー：二重入力ゼロ/集計ゼロ/意思決定が速くなる 等
- CTAボタン2つ + “30秒でわかる”要約（3点）
- 右側：プロダクトの雰囲気が伝わるUIモック（ダミーでも良いが高品質）
  ※本物のスクショがない前提で、上品なダッシュボード風モックを自作

B. 共感（課題）→解決（価値）
- 「ツールが増えるほど、現場が止まる」的な課題を3〜5点（短く）
- BizFloがどう解決するかを対応させる（Before/After）

C. 統合マップ（価値の核）
- 顧客→案件→見積/契約→請求→会計→KPI/稼働 の流れを図解
- “つながるから速い”を一言で

D. ROI/削減効果（入力式）
- 入力：従業員数/時給/毎月の集計時間/二重入力の回数 など（最小3項目）
- 出力：削減時間（試算）/削減コスト（試算）/回収目安（試算）
- 注意書き（※条件により変動）
- このブロックの下にCTA（導入相談）

E. 主要機能（4〜6個）
- CRM / 案件・タスク / 売上・粗利 / KPI / 稼働分析 / 会計（GL/PL/BS/CF）
- “特徴→嬉しいこと”の1セットで書く
- UIカードで見やすく

F. ユースケース（タブ）
- 経営者：意思決定が速い、着地が読める
- 営業責任者：案件パイプライン、確度、受注予定、活動量
- 管理部門：請求〜会計、月次締め、証憑/帳票、ミス削減

G. 競合比較表
- 複数ツール運用（SFA/会計/請求/タスク…） vs BizFlo
- 軸：コスト、データ一貫性、設定/運用負担、経営可視化、拡張性
- “BizFloが勝つ理由”を表の下で一言まとめ

H. 導入フロー（不安解消）
- 最短の流れ：ヒアリング→初期設定→運用開始→伴走改善
- “丸投げではなく伴走で定着”を強調

I. 信頼（セキュリティ/サポート）
- データ管理方針（一般的でOK）
- 権限管理/監査ログなど「ありそうで安心」な項目を誇張せずに
- サポート：導入支援/運用設計/定例/チャット等（過度な断言は避ける）

J. 料金
- スタンダード：月額29,800円（大きく）
- エンタープライズ：要相談（拡張/個別対応）
- “この価格で統合できる”を強調
- 料金セクションにもCTA

K. FAQ（8〜12個）
- よくある反論を先回りして潰す（導入期間、移行、権限、カスタマイズ、サポート、セキュリティ、解約など）

L. 最終CTA（強い締め）
- “まずは10分相談”の雰囲気で心理的ハードルを下げる
- フォーム or モーダルフォーム（入力少なめ）
- 送信後に「次の案内」表示（例：日程調整・資料DL）

# 5) フォーム仕様（CV最大化）
- 入力は最小：会社名 / お名前 / メール / 任意メッセージ（または導入目的選択）
- バリデーション、送信中UI、成功/失敗Toast
- 送信成功時はサンクスメッセージ + 次アクション導線
- envで送信先差し替え可能に（例：LEAD_WEBHOOK_URL）

# 6) 計測（必須）
- イベント計測の雛形を実装（Google Tag Manager想定で dataLayer push）
  - cta_click（どのCTAか）
  - form_open
  - form_submit
  - form_success
- 実際のIDはenvにして空でも動くように

# 7) 品質基準（必須）
- モバイルで最も美しく、最も押したくなる
- 文章は自然で説得力がある（過剰な誇張禁止）
- コンポーネント分割が美しい（sections/ ui/ lib）
- 速度最適化（画像最適化・アニメ控えめ・不要な依存削除）
- アクセシビリティ最低限（ボタン/ラベル/コントラスト）
- “とにかく完成度が高い”状態まで仕上げる（仮の画像やコピーも品質を落とさない）

# 8) 実装指示（出力方法）
- まずディレクトリ構成を提示し、次に各ファイルをフルで出力してください。
- 省略は禁止（主要ファイルは全て内容を出す）。
- 最後に `npm install` から `npm run dev` までの手順をREADME形式で提示。
- 追加で「改善余地（A/B案）」を3つ提案（実装はしなくて良い）。
